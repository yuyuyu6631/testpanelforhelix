# 前端对接接口文档 V5.0

本文档专门为前端开发整理，按页面功能模块划分接口调用。
V5.0 新增了 **API 定义中心 (Interface Template)** 模块。

**Base URL**: `http://localhost:8000` (或配置的后端地址)

---

## 1. 接口定义中心 (Definition Center) [NEW]
*位置: `/templates` 页面*

### 列表与查询
| 功能 | 方法 | 路径 | 参数 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **加载模板列表** | `GET` | `/templates` | `?skip=0&limit=100` | 返回模板数组 |

### 模板操作
| 功能 | 方法 | 路径 | Body |
| :--- | :--- | :--- | :--- |
| **新建模板** | `POST` | `/templates` | `{ "code": "...", "name": "...", "endpoint": "...", ... }` |
| **获取详情** | `GET` | `/templates/{id}` | 无 |
| **编辑模板** | `PUT` | `/templates/{id}` | 完整的模板对象 |
| **删除模板** | `DELETE` | `/templates/{id}` | 无 |

### 调试预览 (Preview Panel)
*用于在编辑页面实时验证模板配置*

- **URL**: `/templates/debug`
- **Method**: `POST`
- **Body**:
```json
{
  "template": { /* 当前编辑的模板对象, 同 Create/Update Body */ },
  "variables": {
    "order_id": "1001",
    "token": "test-token"
  }
}
```
- **Response**:
```json
{
  "request": {
    "url": "http://api.com/v1/orders/1001",
    "headers": {"Authorization": "Bearer test-token"},
    "body": "..."
  },
  "response": {
    "status_code": 200,
    "text": "...",
    "json": {...},
    "duration": 0.5
  }
}
```

## 2. 系统配置 (Config)
*用于全局配置页面或应用初始化加载*

| 功能 | 方法 | 路径 | 参数/Body | 返回值关键字段 |
| :--- | :--- | :--- | :--- | :--- |
| **获取配置** | `GET` | `/config` | 无 | `user_token`, `max_workers` |
| **保存配置** | `POST` | `/config` | `{"user_token": "...", "max_workers": 5}` | 更新后的配置对象 |

## 3. 用例管理页 (Test Cases)
*位置: `/cases` 页面*

### 列表与查询
| 功能 | 方法 | 路径 | 参数 | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **加载用例列表** | `GET` | `/cases` | `?skip=0&limit=100` | 返回用例数组 |

### 用例操作
| 功能 | 方法 | 路径 | Body |
| :--- | :--- | :--- | :--- |
| **新建用例** | `POST` | `/cases` | `{"question": "...", "is_active": true}` |
| **编辑用例** | `PUT` | `/cases/{id}` | 完整的用例对象 |
| **删除单条** | `DELETE` | `/cases/{id}` | 无 |
| **批量删除** | `DELETE` | `/cases/batch` | Body: `[id1, id2, id3]` |
| **清空所有** | `DELETE` | `/cases/clear-all` | 无 |

### 批量操作
| 功能 | 方法 | 路径 | Body |
| :--- | :--- | :--- | :--- |
| **批量启用/禁用** | `POST` | `/cases/batch-status` | `{"case_ids": [...], "is_active": true/false}` |

### 导入与生成 (Action Bar)
| 功能 | 方法 | 路径 | 参数 |
| :--- | :--- | :--- | :--- |
| **上传文件导入** | `POST` | `/cases/import` | FormData: `file` (Excel/CSV) |
| **AI 自动生成** | `POST` | `/generate/cases` | Query: `?count=2` (每指标生成数) |
| **获取生成预览数据** | `GET` | `/generate/preview` | 无 (返回指标数、公司数) |

---

## 4. 测试执行页 (Test Runner)
*位置: `/runner` 页面*

### 执行控制
| 功能 | 方法 | 路径 | Body | 说明 |
| :--- | :--- | :--- | :--- | :--- |
| **开始测试** | `POST` | `/run` | `{"case_ids": []}` | 空数组表示运行所有活跃用例。返回 `batch_id` |
| **停止测试** | `POST` | `/run/stop` | Query: `?batch_id=...` | 强制停止运行中的批次 |

### 状态监控
| 功能 | 方法 | 路径 | 说明 |
| :--- | :--- | :--- | :--- |
| **获取正在运行的批次** | `GET` | `/run/active-batches` | 用于页面刷新后恢复进度条显示 |
| **WebSocket 实时日志** | `WS` | `/run/ws/{batch_id}` | 监听 `update`, `batch_status`, `done` 消息 |

## 5. 报告记录页 (Reports)
*位置: `/reports` 页面*

| 功能 | 方法 | 路径 | 备注 |
| :--- | :--- | :--- | :--- |

## 6. Template 类型定义 (TypeScript Interfaces)

```typescript
interface InterfaceTemplate {
  id: number;
  code: string;       // 唯一标识, 如 "order_create"
  name: string;       // 显示名称
  description?: string;
  version: string;    // "v1.0"
  
  // Request Config
  base_url: string;
  method: "GET" | "POST" | "PUT" | "DELETE" | "PATCH";
  endpoint: string;   // "/api/v1/orders/{id}"
  
  body_type: "json" | "form-data" | "raw" | "none";
  body_template?: string; // Jinja2 string
  query_params?: string;  // JSON string
  headers?: string;       // JSON string
  
  // Auth Config
  auth_type: "none" | "bearer" | "apikey" | "basic" | "custom";
  auth_config?: string;   // JSON string
  
  // Response Config
  response_parser?: string; // JSON string
  
  timeout: number;
  retry_count: number;
  is_active: boolean;
}
```
